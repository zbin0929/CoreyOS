# 5 分钟客户首次接触 Demo 脚本

> 用途：跟跨境电商卖家做 30 分钟会议时，前 5 分钟用什么打中他
> 演示对象：亚马逊 FBA 美国站卖家（不限规模）
> 演示工具：CoreyOS + `cross_border_ecom` Pack v0.2.0

---

## 演示前准备（自己做，对方看不到）

1. 打开 CoreyOS，确认 `cross_border_ecom` Pack 已启用
2. 打开 Home 页，确认 `ad-overview` 卡片显示出来（顶部 KPI + 下方提示 + 3 个按钮）
3. 准备好 [`../../src-tauri/assets/skill-packs/cross_border_ecom/samples/`](../../src-tauri/assets/skill-packs/cross_border_ecom/samples/) 三份 demo CSV 备用（如果对方没带数据，用这个兜底）
4. **关闭其他通知 / 微信弹窗**
5. 屏幕共享分辨率调到对方能看清字

---

## 5 分钟主线（卡时间念稿）

### 第 0:00-0:30 · 一句话讲清这是什么

> "我做了个本地装的 AI 软件，专门帮亚马逊卖家盯广告、库存、差评。
> 数据完全在你电脑里跑，不上云。
> 今天我给你看 3 个场景，你自己判断有没有用。"

**不要讲技术、不要讲架构、不要讲 Pack。**

### 第 0:30-2:00 · 场景 1 · 广告守卫

操作：

1. 在 Home 卡片上点 **「广告日巡检」** 按钮
2. 软件自动跑 workflow，把 demo CSV 的 7 个 campaign 分析一遍
3. 输出会指出：
   - `SP-New-Launch-A14`：花了 18 美金，0 单 → 建议暂停
   - `SP-ASIN-Competitor`：ACOS 55% 远超目标 → 建议降 bid
   - `SP-Brand-Defense`：ACOS 8.35%，CTR 3.23% → 建议加预算

念稿：

> "你看，这就是它干的事 —— 不是给你一张报表让你自己看，
> 是直接告诉你哪个 campaign 该停、哪个该降 bid、哪个该加预算。
> 你每天早上看 5 分钟就够了。"

**最重要的反应**：盯对方眼睛。
- 他点头 / 凑近屏幕 → 中了
- 他面无表情 / 看手机 → 跳过这场景

### 第 2:00-3:30 · 场景 2 · 库存哨兵

操作：

1. 点 **「库存巡检」** 按钮
2. 软件输出 4 档分级：
   - 🔴 断货（3 个 ASIN）
   - 🟡 告警（2 个 ASIN，14 天内会断）
   - 🟢 健康
   - ⚫ 冗余（3 个 ASIN，库龄 > 180 天）

念稿：

> "库存这块也是一样的逻辑。
> 红色的现在就该补货，黄色的两周内补，黑色的赶紧清。
> 你不用每天打开 Sellerboard / 卖家精灵自己挨个看。"

**关键提问**（演示期间问对方）：

> "你现在是怎么盯库存的？多久看一次？"

听他答什么。这是你下一版要改的方向。

### 第 3:30-4:30 · 场景 3 · 差评监控

操作：

1. 点 **「差评巡检」** 按钮
2. 软件按 6 桶分类（质量/物流/描述/功能/服务/其他）+ P0/P1/P2 严重度
3. 例如：B08N 这个 ASIN 7 天内质量类差评比例超阈值 → P0 警报

念稿：

> "差评这块最实用。
> 你每天可能就 2-3 条差评，但哪条该立刻回、哪条可以放一放，分级很重要。
> 它会自动按照'1-2 星 + 质量类 + 最近 7 天'做组合判断，
> 没用的咬文嚼字它自己过滤掉。"

### 第 4:30-5:00 · 关键一问

放下鼠标，看着对方说：

> "你看完这 3 个场景，第一反应是什么？
> 哪个对你最有用？哪个其实没用？"

**然后闭嘴。**

不要补充、不要解释、不要推销。让他说。

---

## 5 分钟之后（剩下 25 分钟用来听）

### 必问的 5 个问题（按顺序）

1. **"你现在每天花多少时间在广告 / 库存 / 差评上？"**
   → 听他报数字，估算他的痛点强度
2. **"现在用什么工具？哪里不爽？"**
   → 听竞品名 + 抱怨点 = 你的差异化机会
3. **"如果今天这个工具立刻能装在你电脑上，你会用吗？"**
   → 听他说"会"还是"再看看"。再看看 = 不会
4. **"如果一个月 X 块（说个数字，比如 200 / 500 / 1000），你愿意付吗？"**
   → 摸价格锚点，**不是真要他签**
5. **"你身边还有 1-2 个同行愿意试吗？"**
   → 病毒系数 + 二次验证

### 不要做的事

- ❌ 不要演示完整产品（设置、配置、其他 Pack 都别碰）
- ❌ 不要讲 Hermes / Tauri / Rust / Pack 架构
- ❌ 不要现场答应做某个功能（"我回去想想能不能做"是标准答案）
- ❌ 不要发 GitHub 链接（开源是后话）
- ❌ 不要让对方现场装软件（远程装机一定卡）

## 演示后 24 小时内（必做）

1. **写一段 100 字反馈纪要**，发到 [`../../docs/status/`](../../docs/status/) 下新建的 `interview-log.md`（每场一条）
2. **挑 1 条对方亲口说"这个有用"的话**，原话记下来 —— 这是产品 tagline 备选
3. **挑 1 条对方亲口说"如果有 XXX 就好了"的话** —— 这是下一版功能候选
4. **第 2 天发感谢**：

```
[名字] 昨天聊得很受益。
我整理了几条你提到的点，一周内会改一版给你看。
不着急，你先该忙啥忙啥。
```

## 演示翻车的常见情况 + 应对

### "我连不上网" / "工具打不开"
→ "今天网络问题，我录屏给你看，你下次开方便时间再来。"
（不要硬撑，体验差比不演示更糟）

### 对方导自己的真实数据进去 → 出错 / 跑得乱
→ "诶这个数据格式我没见过，你能发一份给我吗？这周改完我重新约你。"
（这就是金矿。真实数据格式的差异 = 你下版要做的事）

### 对方明显走神
→ "我看你今天可能忙，要不我们另约？"
（主动放手比硬演完得分高）

### 对方提一堆功能要求
→ "我记下来。但我先问一下，**你现在演示的这 3 个功能**对你有用吗？"
（不要被带跑去做新功能，先确认核心场景）

---

## 成功的演示标志

- 对方至少有 **1 次** 凑近屏幕
- 对方至少问 **1 个** 具体细节问题（不是"价格多少")
- 对方主动问 **"什么时候能用"**
- 对方主动 **介绍另一个朋友** 给你

任意 2 条达成 = 这是你的种子客户。

---

## 演示翻完之后

- **当晚** 把 5 个问题的答案写进 [`./interviews.md`](./interviews.md)（自己建一个，每场一节）
- **第 3 场跑完** 在 [`../status/FOCUS.md`](../status/FOCUS.md) 第 3 件事开始执行
- **不要在第 1、2 场之后立刻改代码**。等满 3 场反馈合并再动。
