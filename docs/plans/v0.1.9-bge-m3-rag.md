## 二、统一下载中心（DownloadCenter）

### 2.1 背景

当前应用更新只有一个小 banner，没有进度百分比，失败后只有一行错误。
BGE-M3 模型 ~1.2GB，没有进度界面用户会以为卡死了。
以后可能还有更多下载需求（语音模型、浏览器引擎等）。

### 2.2 设计

**位置**：Settings → 新增 "下载管理" section，或在侧边栏底部弹出。

**状态机**：

```
idle → downloading → completed / error
                 ↓
              paused（可选，暂不实现）
```

**UI 元素**：
- 下载项名称（如 "BGE-M3 语义模型"、"Corey v0.1.9 更新包"）
- 进度条（百分比 + 已下载/总大小，如 `456.2 MB / 1.18 GB`）
- 下载速度（如 `12.3 MB/s`）
- 状态文字（等待中 / 下载中 / 校验中 / 安装中 / 已完成 / 失败）
- 失败时显示错误信息 + 重试按钮

**Rust IPC**：

```rust
#[tauri::command]
fn download_start(task_id: String, url: String, target_path: String) -> Result<()>;

#[tauri::command]
fn download_cancel(task_id: String) -> Result<()>;

// 事件通过 tauri::Emitter 推送到前端
// event: "download:progress" { task_id, downloaded, total, speed }
// event: "download:completed" { task_id, path }
// event: "download:error" { task_id, message }
```

**应用更新集成**：
- `useAppUpdater` 的下载逻辑改为调用 DownloadCenter
- UpdateBanner 显示进度时从 DownloadCenter 读取状态
- 下载完成后自动触发安装 + relaunch

## 三、BGE-M3 本地 Embedding

### 3.1 模型规格

| 属性 | 值 |
|------|-----|
| 模型 | BAAI/bge-m3 |
| 参数量 | 568M |
| 文件大小 | ~1.2 GB（F16 ONNX） |
| 向量维度 | 1024 |
| 最大上下文 | 8192 tokens |
| 语言支持 | 100+ 语言（中文、英文等） |
| 许可证 | MIT |

### 3.2 下载源

| 源 | URL | 说明 |
|----|-----|------|
| 国内 CDN（主） | 用户提供的国内服务器 | 国内直连，速度有保障 |
| HuggingFace 镜像（备） | `https://hf-mirror.com/baaibge-m3` | hf-mirror.com 国内可达 |

**模型文件存放路径**：
- macOS: `~/.hermes/models/bge-m3/model.onnx`
- Windows: `%USERPROFILE%\.hermes\models\bge-m3\model.onnx`

### 3.3 安装流程

1. 用户在 Settings → Knowledge → "语义增强" 开关打开
2. 弹出确认：需要下载 ~1.2GB 模型文件
3. 用户确认 → DownloadCenter 开始下载
4. 下载完成 → 校验文件 hash → 加载到 ONNX Runtime
5. 成功 → 开关变为"已启用"，开始对已有 Knowledge 文件建索引
6. 失败 → 显示错误，开关回退

### 3.4 Rust 实现

**依赖**：
```toml
# Cargo.toml
[target.'cfg(not(target_arch = "wasm32"))'.dependencies]
ort = { version = "2", features = ["load-dynamic"] }
tokenizers = "0.21"
```

**模块结构**（扩展现有 `ipc/embedding.rs`）：

```rust
// ipc/embedding.rs — 新增部分

struct BgeM3Embedder {
    session: ort::Session,      // ONNX Runtime session
    tokenizer: tokenizers::Tokenizer,
    dim: usize,                  // 1024
}

impl BgeM3Embedder {
    fn load(model_path: &Path) -> Result<Self>;
    fn embed(&self, text: &str) -> Result<Vec<f32>>;      // 单条
    fn embed_batch(&self, texts: &[&str]) -> Result<Vec<Vec<f32>>>;  // 批量
}
```

**恢复之前删除的函数**（参考 `embedding.rs` 顶部的注释）：
- `embed` / `embed_single` — 调用 BgeM3Embedder
- `store_embedding` — 写入 `knowledge_chunks.embedding` BLOB 列
- `search_by_vector` — 余弦相似度搜索
- `cosine_similarity` — 向量距离计算

### 3.5 混合搜索流程

```
用户查询 query
    ↓
expand_query(query) — 同义词扩展
    ↓
    ├── BM25 关键词搜索 → keyword_scores[]
    ├── BGE-M3 向量检索 → vector_scores[]
    ↓
rrf_fuse(keyword_scores, vector_scores) — 倒数秩融合
    ↓
返回排序后的 chunks
```

**RRF 公式**：`score(d) = Σ (1 / (k + rank_i))`，k=60（标准参数）

### 3.6 索引构建

- Knowledge 文件上传时，`chunk_text_smart()` 分块
- 如果 BGE-M3 已启用，每个 chunk 调用 `embed()` 生成 1024 维向量
- 向量存入 `knowledge_chunks.embedding` BLOB 列
- 已有未建索引的 chunks：后台批量补建（启动时 + 新文件上传时）

### 3.7 性能预估

| 操作 | 预估耗时 |
|------|---------|
| 单条 embed（CPU） | ~50ms |
| 批量 embed 100 条 | ~2s |
| 向量搜索 10K chunks | ~20ms（暴力搜索） |
| 首次加载模型 | ~3s |
| 内存占用 | ~1.5 GB（模型 + 运行时） |

如果后续 chunks 超过 50K，考虑引入 HNSW 索引（`instant-distance` 或 `hnsw` crate）。

## 四、Knowledge 页面集成

```
[语义增强] ○ 关闭  ○ 已启用
```

## 六、风险与缓解

| 风险 | 缓解措施 |
|------|---------|
| ONNX Runtime 编译复杂 | 用 `ort` crate 的 `load-dynamic` feature，运行时加载系统库 |
| 模型文件大（1.2GB） | 国内 CDN + 断点续传 + 进度显示 |
| 内存占用高（~1.5GB） | 模型懒加载，不用时不加载；提供卸载选项 |
| Windows 编译 ONNX | CI 已有 Windows runner，提前验证编译 |
| BGE-M3 ONNX 格式 | 使用 HuggingFace `optimum` 导出的官方 ONNX 文件 |
