## 简介

Corey 是一个面向 AI Agent 的控制台应用，专为使用 Hermes Agent 或其他 AI 助手的用户设计。它提供了一个美观、快捷的图形界面，让你无需依赖命令行即可管理 AI 助手的所有功能。

### 主要特点

- **多平台支持**：支持 Telegram、Discord、Slack、微信等 8 个即时通讯平台
- **多模型管理**：统一管理多个 AI 模型供应商和模型
- **成本控制**：实时监控 token 用量和成本，设置预算告警
- **配置管理**：支持多个配置集，可导入/导出配置
- **数据分析**：可视化的使用统计和趋势分析
- **技能编辑**：可视化编辑 Hermes 技能文件
- **轨迹追踪**：时间线方式查看对话和工具调用历史
- **工作流引擎**：可视化编排多步骤 AI 自动化流程
- **浏览器自动化**：AI 控制浏览器，自动抓取网页数据

## 快速开始

### 首次使用

1. **启动应用**
   - 双击 Corey 应用图标启动
   - 首次启动会看到欢迎页面

2. **连接到 Hermes**
   - 如果已安装 Hermes，Corey 会自动检测
   - 在「设置」页面可以配置 Gateway 地址和 API Key
   - 默认 Gateway 地址：`http://127.0.0.1:8642`

3. **开始对话**
   - 点击左侧导航的「对话」进入聊天界面
   - 输入消息，按 `Ctrl+Enter` 或 `⌘+Enter` 发送
   - AI 会实时回复，支持流式输出

## 核心功能

### 对话

**功能说明**：与 AI Agent 进行实时对话的核心界面。

**如何使用**：

1. **创建新会话**
   - 点击「新建会话」按钮
   - 会话会自动保存到本地数据库

2. **发送消息**
   - 在底部输入框输入消息
   - 按 `Ctrl+Enter` / `⌘+Enter` 发送
   - 支持多行输入

3. **选择模型**
   - 在输入框上方可以选择使用的模型
   - 也可以在「设置」中设置默认模型

4. **管理会话**
   - 左侧面板显示所有历史会话
   - 可以重命名、删除会话
   - 支持按适配器筛选会话（多代理模式）

5. **工具调用**
   - AI 调用工具时会显示在消息中
   - 可以查看工具调用的参数和结果

**提示**：
- 按 `Esc` 键可以取消正在进行的流式输出
- 支持复制消息内容
- 消息中的代码块支持语法高亮

### 平台通道

**功能说明**：配置和管理 AI Agent 接入的各个即时通讯平台（Telegram、Discord、Slack、微信等）。

**如何使用**：

1. **进入通道配置**
   - 点击左侧导航的「平台通道」
   - 会显示所有支持的 8 个平台

2. **配置通道**
   - 点击某个平台的「编辑」按钮
   - 填写必要的凭证（如 Bot Token）
   - 配置行为参数（如是否仅在被 @ 时回复）

3. **保存并生效**
   - 点击「保存」按钮
   - 某些通道需要重启 Gateway 才能生效
   - 系统会提示是否立即重启

4. **微信扫码登录**
   - 微信通道支持扫码登录
   - 点击「开启扫码」生成二维码
   - 用微信 App 扫码并在手机上确认
   - Session Token 会自动写入配置

**支持的平台**：
- Telegram
- Discord
- Slack
- WhatsApp
- Matrix
- 飞书 (Feishu)
- 企业微信 (Wecom)
- 微信 (WeChat)

**提示**：
- 不同平台需要的凭证不同，请按提示填写
- 凭证会保存在 `~/.hermes/.env` 文件中
- 可以通过「探测」按钮测试通道连接状态

### 配置集

**功能说明**：管理多个独立的 Hermes 配置目录，用于隔离不同环境（生产/开发/演示）的配置。

**如何使用**：

1. **查看配置集**
   - 点击左侧导航的「配置集」
   - 显示所有已创建的配置集
   - 当前使用的配置集会标记为「使用中」

2. **创建新配置集**
   - 点击「新建配置集」
   - 输入配置集名称（如 `dev`、`prod`）
   - 系统会在 `~/.hermes/profiles/` 下创建对应目录

3. **切换配置集**
   - 点击非激活配置集的「启用」按钮
   - 确认切换操作
   - 可以选择同时重启 Gateway 让变更立即生效

4. **导入配置集**
   - 点击页面右上角的「导入」按钮
   - 选择之前导出的 `.tar.gz` 归档文件
   - 预览归档内容（配置集名、文件列表、导出时间）
   - 可以重命名目标配置集名称
   - 如果目标已存在，会提示是否覆盖

5. **导出配置集**
   - 点击某个配置集的「导出」按钮
   - 系统会生成 `.tar.gz` 归档文件
   - 文件会自动下载到浏览器的下载目录
   - 归档包含完整的配置目录和元数据

6. **克隆配置集**
   - 点击「克隆」可以复制现有配置集
   - 适用于创建基于现有配置的新环境

7. **删除配置集**
   - 点击「删除」按钮
   - 确认后从磁盘删除配置集目录
   - 无法删除当前正在使用的配置集

**提示**：
- 配置集之间完全隔离，模型、API Key、通道配置都独立
- 导出的归档可用于备份或在其他机器上恢复
- 切换配置集后，Gateway 会读取新配置集的设置

### 用量分析

**功能说明**：可视化的使用统计，包括 token 用量、成本趋势、模型分布、工具调用统计等。

**如何使用**：

1. **查看概览**
   - 点击左侧导航的「用量」
   - 顶部显示关键指标：会话数、消息数、工具调用、活跃天数、总 token 数

2. **活动趋势**
   - 「活动」图表显示最近 30 天每天的消息数
   - 帮助了解使用频率

3. **Token 用量**
   - 「Tokens」图表显示最近 30 天的 prompt 和 completion token
   - 帮助监控成本

4. **模型分布**
   - 「Top 模型」显示按会话数排序的模型使用情况
   - 了解最常用的模型

5. **工具统计**
   - 「Top 工具」显示按调用次数排序的工具
   - 了解 AI 最常使用的功能

6. **Agent 统计**
   - 「按 Agent 统计」显示不同适配器的会话分布
   - 适用于多代理模式

**提示**：
- 所有数据存储在本地 SQLite 数据库
- 点击「刷新」可以更新统计
- 数据基于本地会话历史计算

### 日志

**功能说明**：查看 Hermes 的各类日志，包括 Agent 日志、Gateway 日志、错误日志和配置变更日志。

**如何使用**：

1. **选择日志类型**
   - 点击左侧导航的「日志」
   - 顶部有 4 个标签页：
     - **Agent**：Hermes Agent 运行日志
     - **网关**：Gateway 服务日志
     - **错误**：错误日志
     - **变更日志**：配置变更历史

2. **查看日志**
   - 日志实时滚动显示
   - 支持搜索过滤（输入关键字）

3. **配置变更日志**
   - 每次修改配置都会记录
   - 可以查看改前和改后的差异
   - 点击「撤销」可以恢复到之前的配置
   - 某些操作（如删除 API Key）无法撤销

**提示**：
- 日志文件位于 `~/.hermes/logs/` 目录
- 变更日志位于 `~/.corey/changelog.jsonl`
- 可以通过筛选框快速定位特定日志

### 设置

**功能说明**：配置应用的各项设置，包括外观、Gateway、存储路径等。

**如何使用**：

1. **外观设置**
   - 主题：深色 / 浅色 / 跟随系统

2. **Gateway 设置**
   - Base URL：Gateway 服务地址（默认 `http://127.0.0.1:8642`）

   - 点击「测试连接」可以验证配置

3. **默认模型**
   - 设置会话未指定模型时使用的默认模型
   - 单个会话可以在输入框覆盖此设置

4. **存储设置**
   - 显示配置目录、数据目录、数据库路径

   - 配置目录：`~/.hermes/`
   - 数据目录：`~/.corey/`

**提示**：
- 外观设置保存在本地，不影响 Hermes 配置
- Gateway 设置修改后即时生效

## 高级功能

### 多模对比

**功能说明**：同一个 prompt 并行发送给最多 4 个模型，实时对比输出结果。

**如何使用**：

1. **进入对比页面**
   - 点击左侧导航的「多模对比」

2. **选择模型**
   - 点击「添加模型」选择要对比的模型
   - 最多支持 4 个模型同时对比
   - 可以移除已添加的模型

3. **输入 prompt**
   - 在输入框输入要测试的 prompt
   - 支持多行输入

4. **运行对比**
   - 点击「运行」或按 `Ctrl+Enter` / `⌘+Enter`
   - 所有模型会并行运行
   - 每个模型的输出显示在独立的泳道中

5. **查看结果**
   - 系统会标注本轮"赢家"：最快响应、Token 最多
   - 可以点击「全部停止」中断所有运行
   - 支持导出为 Markdown 或 JSON

**提示**：
- 适合用于模型选型和 prompt 优化
- Token 消耗会按模型数量累加
- 可以取消单个模型的运行

### 运行手册

**功能说明**：创建带占位符的 prompt 模板，可以在命令面板或此页面快速调用。

**如何使用**：

1. **进入运行手册**
   - 点击左侧导航的「运行手册」

2. **创建运行手册**
   - 点击「新建」
   - 填写名称和描述
   - 在模板中使用 `{{参数名}}` 定义占位符
   - 系统会自动检测模板中的参数

3. **设置作用域**
   - **所有配置集**：通用运行手册，所有配置集可用
   - **仅当前配置集**：只在特定配置集下可用

4. **使用运行手册**
   - 在此页面点击「使用」
   - 填充参数值
   - 点击「发送到 Chat」
   - 或在命令面板（`⌘K`）中搜索并调用

**示例模板**：
```
请帮我写一个 {{编程语言}} 的 {{功能类型}} 函数，
要求：{{具体要求}}
```

**提示**：
- 适合复用常用的 prompt 结构
- 参数化使得同一个模板可以用于不同场景
- 作用域控制让运行手册更灵活

### 预算管理

**功能说明**：设置成本预算，按范围、周期、超额动作限制花费，避免意外超支。

**如何使用**：

1. **进入预算页面**
   - 点击左侧导航的「预算」

2. **创建预算**
   - 点击「新建预算」
   - 配置以下参数：
     - **范围**：全局 / 模型 / 配置集 / 适配器 / 通道
     - **范围值**：具体的模型 ID、配置集名称等
     - **金额**：预算金额（美元）
     - **周期**：每日 / 每周 / 每月
     - **超额动作**：通知 / 阻断 / 通知+阻断

3. **监控预算**
   - 预算列表显示当前使用状态
   - 达到 80% 时会显示警告
   - 超过预算时会触发配置的动作

4. **编辑预算**
   - 点击「编辑」修改预算参数
   - 保存后立即生效

**提示**：
- 基于本地使用历史和内置价格表计算
- 阻断模式下，超预算后无法发送新消息
- 适合控制成本，避免意外超支

### 定时任务

**功能说明**：按 cron 计划自动向 Hermes 发送提示词，实现定期执行（每日汇报、定时抓取数据、自动清理等）。

**如何使用**：

1. **进入定时任务**
   - 点击左侧导航的「定时任务」（`⌘8` / `Ctrl+8`）

2. **新建任务**
   - 点击右上角「新建任务」
   - **名称**：任务显示名（如"每日站会总结"）
   - **Cron 表达式**：6 字段形式 `秒 分 时 日 月 周`
     - 示例：`0 0 9 * * *` = 每天 09:00
     - 示例：`0 */5 * * * *` = 每 5 分钟
     - 示例：`0 30 18 * * MON-FRI` = 工作日每天 18:30
   - **提示词**：触发时作为单次对话发给 Hermes 的文本
   - **启用**：勾选后立即生效，可随时在列表中暂停
   - 输入 cron 后下方会实时显示下次触发时间作为校验

3. **管理任务**
   - **暂停/恢复**：点击卡片上的 ▶/⏸ 图标，不丢失定义
   - **编辑**：修改名称、表达式、提示词
   - **删除**：彻底移除
   - 每个任务会显示上次运行时间和结果（✓ 成功 / ✗ 失败），失败时展示错误信息

**行为说明**：
- 每次触发使用 `chat_once`（非流式）调用 Hermes，不会在聊天页新建会话——纯后台执行
- 仅 Hermes 适配器支持定时任务（Claude Code、Aider 暂不支持）
- 应用关闭期间错过的触发**不会补发**（这是 cron，不是任务队列）
- 应用启动时自动载入并调度所有启用的任务
- 任务和执行状态存储在本地 SQLite（`~/Library/Application Support/com.caduceus.app/caduceus.db`）

**已推后的功能**：
- 自然语言 → cron 翻译（当前需手写表达式）
- 每任务的完整运行历史表（目前仅保留最后一次结果）

## 常见问题

### Q: 如何连接到 Hermes？

A: 首次启动时，Corey 会尝试自动检测本地 Hermes。如果检测失败，请：
1. 确保 Hermes 已安装并运行
2. 在「设置」中确认 Gateway 地址为 `http://127.0.0.1:8642`
3. 点击「测试连接」验证配置

### Q: 配置集之间如何切换？

A: 在「配置集」页面，点击目标配置集的「启用」按钮，确认后即可切换。建议同时重启 Gateway 让变更立即生效。

### Q: 如何备份配置？

A: 在「配置集」页面，点击某个配置集的「导出」按钮，会生成 `.tar.gz` 归档文件。可以在其他机器上通过「导入」功能恢复。

### Q: Token 用量如何计算？

A: Token 用量基于本地会话历史和内置价格表计算。在「用量」页面可以查看详细的 token 统计和成本估算。

### Q: 如何设置预算告警？

A: 在「预算」页面创建预算规则，设置范围、周期、金额和超额动作。达到 80% 时会警告，超过预算时会触发通知或阻断。

### Q: 支持哪些即时通讯平台？

A: 支持 8 个平台：Telegram、Discord、Slack、WhatsApp、Matrix、飞书、企业微信、微信。在「平台通道」页面配置。

### Q: 数据存储在哪里？

A:
- Hermes 配置：`~/.hermes/`
- Corey 数据：`~/.corey/`
- 在「设置」的「存储」部分可以查看完整路径

### Q: 如何撤销配置更改？

A: 在「日志」页面的「变更日志」标签中，可以看到所有配置更改历史。点击「撤销」可以恢复到之前的配置（某些操作无法撤销）。

### Q: 多代理模式如何使用？

A: 在顶部栏的 Agent 切换器中选择不同的适配器（Hermes、Claude Code、Aider 等）。每个适配器有独立的能力和会话。

### Q: 应用支持哪些语言？

A: 目前支持中文和英文。在「设置」的「外观」部分可以切换语言。

## 工作流

工作流引擎让你通过可视化拖拽编排多步骤 AI 自动化流程。

### 节点类型

| 节点 | 图标 | 说明 |
|------|------|------|
| Agent | 🤖 | AI Agent 执行任务 |
| Tool | 🔧 | 调用工具 |
| Browser | 🌐 | 浏览器自动化（抓取网页数据） |
| Parallel | ⚡ | 并行执行多个分支 |
| Branch | 🔀 | 条件分支 |
| Loop | 🔄 | 循环执行 |
| Approval | ✋ | 人工审批 |

### 使用方式

1. 侧边栏点击「工作流」进入列表页
2. 点击「新建工作流」或选择一个预置模板
3. 在编辑器中拖拽节点、连线建立依赖关系
4. 点击节点编辑属性（Prompt、Agent、URL 等）
5. 点击「运行」执行工作流
6. 实时查看每个步骤的执行状态

### 预置模板

- **每日新闻摘要** — 定时搜索新闻并生成摘要
- **AI 漫画流水线** — 搜索热门视频 → 分析 → 创作 → 审核
- **代码审查流水线** — PR 提交后自动审查
- **抖音热门视频抓取** — 搜索关键词提取视频信息
- **竞品价格监控** — 并行抓取京东+淘宝价格对比
- **UPS 物流追踪** — 查询包裹状态，自动保存登录态

### 人工审批

当工作流执行到审批节点时会暂停，等待你点击「通过」或「拒绝」。

## 浏览器自动化

工作流中的 Browser 节点能让 AI 控制真实浏览器，自动完成网页操作。

### 配置 LLM

浏览器自动化需要一个 AI 模型来理解网页内容。在「设置」→「浏览器自动化 LLM」中配置：

- **模型**：推荐 DeepSeek Chat（便宜）或本地 Ollama（免费）
- **API Key**：对应模型的 API 密钥
- **Base URL**：留空用默认，Ollama 填 `http://localhost:11434/v1`

### Cookie 持久化

Browser 节点的「浏览器配置文件」字段可以让多个步骤共享登录态：

1. 给需要登录的 Browser 节点设置相同的 Profile 名称（如 `ups-session`）
2. 第一次运行时登录
3. 后续运行自动复用 Cookie，无需再次登录

### 支持的网站

- ✅ 公开页面（新闻、价格查询等）— 直接抓取
- ⚠️ 需要登录的网站 — 配合 Cookie 持久化
- ⚠️ 有验证码的网站 — 简单验证码可以，复杂的需人工介入
