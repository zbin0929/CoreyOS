# AI 浏览器 — 安全与权限

> 客户演示时的「客户问 X，你答 Y」手册。所有回答都基于 v0.2.12 已落地的实际能力，
> 没有 vapor-ware。

## 一、客户最常问的 5 个问题

### Q1：AI 的权限有多大？能操作哪些？不能操作哪些？

**短答**：AI 在专属 Chrome 里能做的事 ≈ 一个新员工开你的浏览器能做的事。**全部受
3 层管控**：

| 层 | 谁负责 | 作用 |
|----|--------|------|
| L1 登录权 | **你** | AI 不知道你的密码。每个系统只有你登过它才有权限。退出登录 = 立刻断权。 |
| L2 行为约束 | **AI 的 soul prompt（系统提示）** | 写操作（点击/填表/提交）必须先问你，等你确认才执行。 |
| L3 审计 | **Corey 日志系统** | 每一次工具调用都有时间戳 + 参数 + 结果，永久写入 \`~/.hermes/logs/\`，Settings → Logs 可查。 |

### Q2：AI 万一自作聪明删了东西怎么办？

**短答**：不会"自作聪明"——L2 约束写在 base soul 里（CoreyOS 内置的 L1 系统提示），
**所有可能改服务器状态的操作都必须先问你，等你按"确认"再执行**。

具体被守护的关键词覆盖：

- 删除/移除/清空（Delete / Remove / Clear / Drop / Discard）
- 取消/退订/退款（Cancel / Refund / Unsubscribe / Revoke）
- 下单/支付（Place Order / Pay / Checkout / Confirm Payment）
- 发送/发布/审批（Send / Publish / Post / Submit / Approve / Reject）
- 改密/改邮箱/改权限（Change Password / Update Email / Grant / Revoke）

AI 看到 \`browser_snapshot\` 返回的按钮文字命中任何一类，**先弹决策卡片**：

```
🧠 我看到的：订单 #A123，状态"待付款"，金额 ¥320
💡 我建议：点击"删除订单"按钮取消这单。
   **这一步会改服务器数据，删了就没了**。
👇 [确认删除]   [先不动]
```

你不点确认，AI 不会动手。

**如果你自己明确说**"删订单 A123"——这已经是 explicit consent，AI 不会再多问一次
浪费你时间。但如果说法模糊（"处理一下这个"），AI 必须先问。

### Q3：要不要把账号密码存在 Corey 这边，让 AI 自己登录？

**短答**：**不需要，也不建议**。CoreyOS **拒绝**这个设计，原因有 4 个：

1. **法律责任** —— 客户的账号密码存在 Corey 这边，被偷了是 CoreyOS 的法律问题。
   《数据安全法》《个保法》明确规定。
2. **多因素认证根本绕不过** —— ERP / 后台普遍强制 MFA + 验证码 + 企业 SSO，
   光有用户名密码也登不上。
3. **Chrome 已经做了** —— 你在 AI 浏览器里登录时，Chrome 弹"是否保存密码？"，
   点保存 → Chrome 用**系统 Keychain (macOS) / DPAPI (Windows)** 加密存。下次自动
   填充，AI 看到的就是已经登录的页面。
4. **行业实践** —— Cursor / ChatGPT Browser Mode / Browser Use 都不存账密。

**演示话术**：

> "AI 浏览器是一个完整的 Chrome——它自带密码管理器。你登录时点保存，密码进系统
> Keychain（OS 级加密）。这比我们自己存账密**安全 100 倍**，而且不绑定 CoreyOS：
> 即便有一天你不用我们，密码也还在你电脑的 Keychain 里。"

### Q4：登录态保存多久？我换电脑会丢吗？

**短答**：在同一台电脑上**永久保留**（直到你手动清除）。换电脑要重新登录。

技术细节：

- AI 浏览器是一个独立 Chrome profile，目录在 \`~/.hermes/chrome-debug/\`（Mac/Linux）
  或 \`%USERPROFILE%\\.hermes\\chrome-debug\\\`（Windows）
- 所有 cookie / localStorage / saved passwords 全部存在这个目录
- Corey 升级 / 重启 / 重启电脑 → 全部保留
- 卸载 Corey / 清空 Hermes 数据目录 / Settings 里点"清除登录态" → 全部清空
- 备份这个目录到云盘 = 备份所有登录态（不建议——里面有 saved password 数据库）

### Q5：怎么撤销 AI 对某个网站的访问权？

**Settings → AI 浏览器 → 已登录的网站** 里每个域名后面都有一个 ✕ 按钮：

- 点 ✕ → 只清除这一个域名的 cookie（其他不动）
- "清除登录态"按钮 → 清空整个 profile（所有域名）

或对话操作：

- "清掉 amazon.com 的登录" → AI 调 \`corey_browser_clear_domain\`
- "忘掉所有登录" → AI 调 \`corey_browser_clear\`

被清除后，下次 AI 访问该域名会落到登录页，需要你重新登录一次。

## 二、内部审计能力（演示时打开给客户看）

打开 **Settings → 日志（Logs）页**，演示三个东西：

1. **工具调用记录** —— 每次 \`browser_navigate\` / \`browser_click\` / \`browser_type\`
   都有一条 entry，时间戳到秒，参数 (URL / ref / 文字) 全部记下来。
2. **审批历史** —— \`approval/respond\` 事件，记录每个决策卡片用户选择了什么、
   什么时候选的。
3. **变更日志** —— \`~/.hermes/.corey/changelog.json\` 记录了 \`BROWSER_CDP_URL\` 的
   写入时间和值（"AI 浏览器什么时候被启用的"）。

> 你可以告诉客户：
>
> "审计是合规的底盘。AI 做的每一步都有日志留底，半年后你想知道'2026 年 5 月 12 日
> 谁删了订单 A123'，翻 logs 一目了然。Corey 是**审计友好**的——不像 SaaS，操作
> 痕迹在我们服务器上你看不到——这边日志全部在你机器上、你拥有、你可备份。"

## 四、演示时一定要做的 3 件事

1. **先在 Chrome 里登录 1-2 个真实系统**（Amazon Seller Central / 你的内部 ERP），
   不然演示「AI 用我的登录」就空口无凭。
2. **演示完读操作后**，故意让 AI 做一个写操作（如"取消订单 A123"），让客户**亲眼
   看到决策卡片弹出**——这是信任建立的关键时刻。
3. **打开一次 Logs 页**——客户对"AI 黑盒"焦虑就靠这个治。
