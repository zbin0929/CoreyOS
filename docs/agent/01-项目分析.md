## 二、技术架构

Tauri 2 + React 18 混合架构，单进程，无独立后端。

### 前端技术栈
- Vite 5 + React 18 + TypeScript (strict)
- TanStack Router + Query
- Zustand (UI 状态)
- shadcn/ui + Tailwind CSS 3
- cmdk (命令面板) + Framer Motion
- react-virtuoso (虚拟列表) + xterm.js (终端) + CodeMirror 6 (编辑器)
- react-i18next (中英双语)

### 后端技术栈 (Rust)
- Tauri 2 + tokio
- reqwest + eventsource-stream (HTTP + SSE)
- rusqlite (bundled SQLite)
- portable-pty (终端)
- serde / serde_json / serde_yaml

## 三、核心架构设计

### AgentAdapter 抽象
- `AgentAdapter` trait 定义统一的 Agent 接口
- `AdapterRegistry` 支持运行时注册/注销/热切换
- 已注册: HermesAdapter (主力) + ClaudeCodeAdapter (mock) + AiderAdapter (mock)
- 支持多 Hermes 实例 + LLM Profile 桥接

### IPC 命令体系
- 80+ 个 IPC 命令覆盖全部功能模块
- 前端 TS 类型手工镜像 Rust 端

### 数据流
- 聊天: IPC → Rust POST → SSE → Tauri events → Zustand → 虚拟渲染
- 配置: 表单验证 → IPC → 原子写入 (tmpfile + rename + fsync) → diff 返回
- 终端: pty_spawn → shell → 双向事件中继
